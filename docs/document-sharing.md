# Document Sharing

This document covers the share-link system for files in StudyBudd.

## Summary

Share links are generated by the backend and enforced with server-side access control.

- `document_shares` stores link metadata (`share_token`, owner, expiry, revoke flag)
- `document_share_recipients` stores optional allowlisted recipient emails
- shared file access is checked on every metadata/download request

## API Endpoints

### Create share link

`POST /api/documents/{document_id}/share-links`

Request body:

```json
{
  "recipient_emails": ["alice@example.com", "bob@example.com"],
  "expires_in_hours": 72
}
```

Response:

```json
{
  "id": "uuid",
  "document_id": "uuid",
  "share_token": "token",
  "share_url": "http://localhost:3000/shared/token",
  "recipient_emails": ["alice@example.com", "bob@example.com"],
  "expires_at": "2026-02-27T12:00:00+00:00",
  "created_at": "2026-02-24T12:00:00+00:00"
}
```

### Resolve shared document

`GET /api/documents/shared/{share_token}`

Returns document metadata if access is allowed.

### Download shared document

`GET /api/documents/shared/{share_token}/download`

Returns file bytes (with the original filename in `Content-Disposition`) if access is allowed.

## Access Control Rules

Access is granted only if all checks pass:

1. Share token exists and is not revoked
2. Share token is not expired (if `expires_at` is set)
3. User passes identity check:
   - owner always allowed
   - if recipients exist, JWT email must be in recipient list
   - if recipients are empty, any authenticated user with token is allowed

## Frontend Integration

`/dashboard/files` share modal now:

- collects recipient emails
- calls `POST /api/documents/{id}/share-links`
- copies the returned `share_url`

Shared links open this page:

- `/shared/[token]`

This page resolves metadata and downloads through the protected backend endpoints.
